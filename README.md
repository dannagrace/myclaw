# myclaw

这是一个 OpenClaw 助手工作区仓库，用来保存助手的人设、协作约定和记忆文件。

## 仓库用途

这个仓库不是传统意义上的应用代码仓库，而是一个 **agent workspace**：

- 保存助手身份设定
- 保存与用户的长期协作约定
- 保存长期记忆与每日记录
- 保存工作区内的行为说明

它的作用更像是给助手提供一个可版本化、可追踪、可恢复的“人格与工作上下文”。

## 目录与文件说明

### 根目录核心文件

- `SOUL.md`
  - 助手的核心行为准则、语气和边界
  - 可以理解为“人格说明书”

- `USER.md`
  - 关于用户的基本信息与协作偏好
  - 用于帮助助手更贴近真实使用习惯

- `IDENTITY.md`
  - 助手自身身份信息
  - 包含名字、定位、风格、emoji 等

- `AGENTS.md`
  - 工作区操作约定
  - 包括启动时读取哪些文件、何时写记忆、哪些事情要谨慎处理等

- `TOOLS.md`
  - 本地环境备注
  - 用于记录技能之外、只对当前环境有效的信息

- `HEARTBEAT.md`
  - 心跳轮询时的任务清单
  - 如果为空，表示不主动做周期性检查

- `MEMORY.md`
  - 长期记忆
  - 存放稳定、值得长期保留的事实与偏好

- `README.md`
  - 当前仓库说明文件

## memory/ 目录

`memory/` 用于保存按日期组织的每日记录，例如：

- `memory/2026-03-13.md`

这些文件更像“工作日志 / 日记流”，记录当天发生的关键事情。

建议：
- 日常事件先写入每日文件
- 稳定、长期有效的信息再提炼进 `MEMORY.md`

## 为什么忽略 `.openclaw/`

`.openclaw/` 已加入 `.gitignore`，原因是它通常存放运行时状态，而不是适合长期版本管理的内容，例如：

- 临时状态
- 会话运行痕迹
- 本地缓存或机器相关信息

这些内容：
- 经常变化
- 可移植性差
- 容易污染提交历史

所以仓库默认只追踪真正有长期价值的文档与记忆文件。

## 当前约定

当前仓库保持以下原则：

- 保持目录结构稳定
- 优先提交有长期价值的 `.md` 文档
- 忽略运行时状态目录 `.openclaw/`
- 重要改动通过 git 提交保留历史

## 使用建议

如果后续继续使用这个工作区，推荐保持下面的节奏：

1. 调整人格/协作方式时更新 `SOUL.md`、`USER.md`、`IDENTITY.md`
2. 每天的重要事件写入 `memory/YYYY-MM-DD.md`
3. 定期把长期有效的信息整理进 `MEMORY.md`
4. 对工作区规则变更时同步更新 `AGENTS.md` 或 `TOOLS.md`

这样这个仓库会越来越像一个可维护、可恢复、可协作的 agent 操作基座。
